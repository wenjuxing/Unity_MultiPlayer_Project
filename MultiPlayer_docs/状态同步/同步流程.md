#### 角色移动(客户端本地预测)

游戏对象身上挂载HeroController的组件主要实现角色移动和改变动画参数；
通过CharacterController.Move()方法客户端先行移动，然后再同步到服务器中；

#### 发送同步请求（客户端）

- 游戏对象身上挂载GameEntity组件用于实现网络同步，使用协程提供同步方法每秒调用10次；

``` csharp
IEnumerator SyncRequest()
    {
        //记录上次发送的状态
        EntityState _lastState = EntityState.None;
        while (true)
        {
            if (IsMine&&transform.hasChanged&&!actor.IsDeath)
            {
                //把本地Vector3转换为网络Vector3
                res.EntitySync.Entity.Id = entityId;
                //角色状态发生改变向服务器发送新的状态
                if (_lastState!= entityState)
                {
                    res.EntitySync.State = entityState;
                    _lastState = entityState;
                }
                this.actor.Position = this.position * 1000;
                this.actor.Direction = this.direction * 1000;
                SetValueTo(this.actor.Position, res.EntitySync.Entity.Position);
                SetValueTo(this.actor.Direction, res.EntitySync.Entity.Direction);
                //发送同步消息
                NetClient.Send(res);
                transform.hasChanged = false;
                //每次发送完状态则清空在res中的状态
                res.EntitySync.State = EntityState.None;
            }
            yield return new WaitForSeconds(0.1f);
        }
    }

```

1. 同步之前会先判断状态是否发生了改变，避免不必要的网络请求浪费带宽；
2. 设置新的位置、旋转以及角色动画状态(站立/奔跑/攻击)，设置Vector时需要先乘以1000，因为在protobuf中Vector是Int32类型的，但是UnityVector3是浮点类型的，当收到同步数据再除以1000还原数据；

#### 处理同步请求（服务端）

```csharp
private void _SpaceEntitySyncRequest(Connection conn, SpaceEntitySyncRequest msg)
        {
            //获取当前角色所在的地图
            var space = conn.Get<Session>().space;
            if (space == null) return;

            //请求同步的信息
            NetEntity netEntity = msg.EntitySync.Entity;
            //服务器中的实际位置
            Entity serEntity = EntityManager.Instance.GetEntity(netEntity.Id);
            //计算位置差
            float dict=Vector3Int.Distance(netEntity.Position,serEntity.Position);
            //使用服务器速度
            netEntity.Speed = serEntity.Speed;
            //限制最大时间差 防止长时间挂机不动后再次移动的移动距离很长
            float dt = Math.Min(serEntity.PositionTime,1);
            //计算限额
            float limit = serEntity.Speed * dt* 1.5f;
            if (float.IsNaN(dict) ||limit<dict)
            {
                SpaceEntitySyncResponse resp = new SpaceEntitySyncResponse();
                resp.EntitySync = new NEntitySync();
                resp.EntitySync.Entity = serEntity.EntityData;
                resp.EntitySync.Force = true;
                conn.Send(resp);
                return;
            }

            //更新需要同步消息
            space.UpdateEntity(msg.EntitySync);
        }
    }
```

1. 先获取服务器中之前的位置，通过Vector3.Distance()计算移动了多少距离；
2. 然后计算当前时间到上次更新时间的时间差，通过速度乘时间差得出这段时间所走的路程；
3. 如果客户端发送的位置是否在允许的误差范围内，则广播给其他客户端；否则的话就把服务器计算的结果作为权威发送给移动的客户端让他回滚到正确的位置；

#### 同步位置和旋转（其他客户端）

```csharp
 private void Update()
    {
        if (!IsMine)
        {
            var targetRotation = Quaternion.Euler(direction);
            this.transform.rotation = Quaternion.Lerp(transform.rotation, targetRotation, Time.deltaTime * 10f);
            Move(Vector3.Lerp(transform.position, position, Time.deltaTime * 10f));
        }
        else
        {
            this.position = transform.position;
            this.direction = transform.rotation.eulerAngles;
        }
    }
```

- 如果不是自己则通过Vector3.Lerp()插值运算平滑同步位置和Quaternion.Lerp()平滑同步旋转；
